## Learnings

- Initial role: Lead for scope, priorities, architecture, and review gates.
- Multi-page static site (not SPA) maximizes SEO, clarity, and maintainability for a product-showcase site.
- Reusable product page template (hero → problem → solution → features → workflow → results → FAQ → CTA) allows rapid iteration on future products while maintaining design consistency.
- WhatsApp-first CTA strategy aligns with both company offering and market expectation (Brazil B2B/SME).
- Portfolio structure (`/portifolio/{slug}`) enables unlimited product additions without rearchitecting.
- Success criteria must balance clarity (10s rule), credibility (real metrics), and conversion (WhatsApp clicks).
- No CMS or backend required initially; static HTML allows rapid iteration and zero infrastructure cost.
- Scope boundaries: architecture defines pages and templates; copy (Switch), visuals (Neo), and implementation (Trinity) are separate domain responsibilities.
